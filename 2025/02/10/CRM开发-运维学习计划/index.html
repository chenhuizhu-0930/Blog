🚀 Dynamics 365 CRM 开发 & 运维学习计划

最近由于项目需要，让ChatGPT给我生成了一份CRM开发运维学习计划。

目标

✅ 能够独立运维 Dynamics 365 CRM 系统，包括插件、工作流、批处理、Web 界面
✅ 掌握 ASP.NET Web Forms、C#、CRM API、日志分析、CI/CD
✅ 具备完整的 CRM 开发与运维能力

📅 总时长：3~6 个月

  • 全职学习（每天 3~5 小时）：3 个月内掌握
  • 兼职学习（每天 1~2 小时）：6 个月掌握

🟢 第一阶段：ASP.NET 基础（2~4 周）

✅ 目标：掌握 ASP.NET Web Forms & MVC，了解 Web 前后端交互，能修改 Web 端代码

1.1 了解 ASP.NET Web Forms

你的项目Web目录包含 .aspx页面（如CostCodeAllot.aspx），需要掌握：

  • Page 事件生命周期（Page_Load, PostBack）
  • ViewState & Session
  • MasterPage & UserControls
  • Web.config 配置（数据库、日志等）
  • 前后端交互（AJAX、jQuery）

📚 学习资料：

  • ASP.NET Web Forms 教程（微软官方）
  • ASP.NET Web Forms 入门（W3Schools）

💡 实战练习：
✅ 修改 Web/View/CostCodeAllot.aspx 页面，增加一个按钮，点击后弹出 Hello CRM!
✅ 在 Web/bin/ 目录找到 Microsoft.Xrm.Sdk.dll，在 Web 界面调用 CRM 数据

1.2 了解 ASP.NET MVC

你的项目主要是 Web Forms，但 MVC 在 CRM 开发中也常见，建议掌握：

  • MVC（Model-View-Controller）架构
  • Razor 语法
  • Web API 调用
  • Dependency Injection (DI)

📚 学习资料：

  • ASP.NET MVC 教程（微软官方）
  • ASP.NET MVC 入门（B 站视频）

💡 实战练习：
✅ 在 Web/BC/Utils.cs 里写一个 string GetMessage() 方法，返回 "CRM is great!"，然后在 Web/View/CostCodeAllot.aspx.cs 里调用并显示

1.3 数据库交互

你的 Web 项目可能调用数据库，需要掌握 CRUD 操作：

  • Entity Framework（ORM 框架）
  • LINQ 查询
  • 数据库连接（ADO.NET）

📚 学习资料：

  • Entity Framework 官方教程                                                   
  • C# 数据访问（微软文档）

💡 实战练习： ✅ 在 Web/BC/Utils.cs 里写一个方法，查询 account 实体并返回名称列表
✅ 在 Web/View/CostCodeAllot.aspx.cs 里调用该方法，并在页面显示数据

🟠 第二阶段：Dynamics 365 CRM 开发（6~8 周）

✅ 目标：掌握 CRM 插件、工作流、批处理，理解 CRM 开发模式

2.1 CRM SDK 基础

你的 Common/ 目录包含 Microsoft.Xrm.Sdk.dll，CRM 开发核心依赖：

  • CRUD 操作（Retrieve, RetrieveMultiple, Create, Update, Delete）
  • 查询表达式（QueryExpression, FetchXML）
  • 调用 CRM Web API

📚 学习资料：

  • CRM SDK 官方文档
  • C# 使用 Xrm SDK 进行 CRUD 操作

💡 实战练习： ✅ 在 Common/BC/OrganizationQueryableContext.cs 里写一个方法，查询 CRM contact 实体
✅ 在 Web/View/CostCodeAllot.aspx.cs 里调用，并在页面显示

2.2 插件开发（Plugins）

你的 Plugin/ 目录包含多个 CRM 插件（如 GetUserPrivileges.cs），建议掌握：

  • IPlugin 机制
  • Pre-Operation / Post-Operation / Asynchronous
  • 注册 & 调试

📚 学习资料：

  • CRM Plugin 开发入门
  • 插件注册工具（Plugin Registration Tool）

💡 实战练习： ✅ 修改 Plugin/GetUserPrivileges.cs，让它在创建 contact 记录时写日志
✅ 使用 Plugin Registration Tool 注册插件，并测试

🔵 第三阶段：运维（4~6 周）

✅ 目标：掌握 CRM 维护、调试、部署、日志分析

3.1 日志分析

📚 学习资料：

  • log4net 配置指南
  • ASP.NET 中使用 log4net

💡 实战练习：
✅ 修改 log4net.xml，让日志输出到本地 log.txt
✅ 在 Web/BC/Utils.cs 里写 Log(string message) 方法
